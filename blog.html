Azure Troubleshooting Cue Cards — Common Issues

Compact study guide with quick steps, ITIL & Agile notes, and a printable cram sheet.

1. What is Azure Active Directory, and how does it differ from on-premises AD?
  1. Azure AD is a cloud-based identity and access management service.
  2. On-prem AD manages devices and users locally using domain join, GPO, Kerberos.
  3. Azure AD supports SSO, MFA, Conditional Access, identity governance.
ITIL: Identity impacts availability & access Agile: iterative policy rollouts
2. How do you troubleshoot login issues in Azure AD?
  1. Check user state: active, not blocked, password validity.
  2. Use sign-in logs to identify authentication or MFA failures.
  3. Review Conditional Access policy assignments.
  4. Validate device compliance in Intune.
  5. Check Azure AD Connect for sync errors.
  6. Document and communicate resolution.
3. Explain Azure Resource Groups and why they are important
  1. Logical containers for Azure resources.
  2. Enable RBAC, lifecycle control, cost management.
  3. Support tagging, automation, structured deployments.
4. How do you monitor Azure resources effectively?
  1. Azure Monitor dashboards and alerts.
  2. Log Analytics for logs & trend analysis.
  3. Application Insights for app telemetry.
  4. Network Watcher for flow & connectivity monitoring.
  5. Integrate alerts with ITSM for automated incidents.
5. Troubleshooting a failing Azure VM
  1. Check VM state (Running/Stopped).
  2. Check Boot Diagnostics for errors.
  3. Review CPU, RAM, IOPS metrics.
  4. Check NSG/firewall/IP config.
  5. RDP/SSH and inspect OS logs.
  6. Redeploy or restore from snapshot if needed.
6. Implement Azure RBAC
  1. Identify least-privilege role needed.
  2. Assign at correct scope (sub/resource group/resource).
  3. Use groups, avoid individual assignments.
  4. Test permissions.
  5. Review regularly for stale permissions.
7. Troubleshoot Azure Storage issues
  1. Check storage health and service status.
  2. Validate firewall/VNet/private endpoints.
  3. Verify keys/SAS/RBAC permissions.
  4. Check for throttling/quota limits.
  5. Test across clients & review logs.
8. Troubleshoot Azure Backup failures
  1. Check Recovery Services Vault job status.
  2. Review backup logs and error codes.
  3. Validate storage and network connectivity.
  4. Test restore to confirm integrity.
  5. Adjust retention or schedule if needed.
9. Troubleshoot Azure Networking
  1. Check IP, VNet, NSG, UDR configuration.
  2. Use Network Watcher tools.
  3. Ping/tracert + flow logs.
  4. Fix misconfigured rules or routes.
10. Troubleshoot hybrid Azure AD sync issues
  1. Check Azure AD Connect health & sync service.
  2. Identify missing/duplicate attributes.
  3. Fix AD objects or filtering.
  4. Re-run sync and confirm.
11. Troubleshoot Conditional Access / MFA issues
  1. Identify impacted apps/users.
  2. Review sign-in logs for policy blocks.
  3. Verify device compliance.
  4. Reset MFA or adjust policy.
12. Troubleshoot VM performance issues
  1. Review CPU/RAM/IOPS metrics.
  2. Inspect OS resource usage.
  3. Optimize apps & disk.
  4. Scale VM size if necessary.
13. Troubleshoot Azure SQL connectivity
  1. Check DB/server status.
  2. Validate firewall/VNet rules.
  3. Test from SSMS/Azure Data Studio.
  4. Fix routing/IP/credentials issues.
14. Prevent Azure outages proactively
  1. Build dashboards & automated alerting.
  2. Implement redundancy & geo-replication.
  3. Review RBAC and policies regularly.
  4. Test backups + DR plans.
  5. Perform post-mortems and improve runbooks.
RMM 1. How do you manage, tune, and optimize RMM alerts to reduce noise?
  1. Identify high-volume alerts and categorize by severity (Critical, Warning, Informational).
  2. Adjust thresholds (CPU/memory/disk/latency) to reflect real-world patterns.
  3. Suppress non-actionable alerts and convert them into informational reports.
  4. Create alert dependencies (e.g., suppress server alerts during maintenance windows).
  5. Implement automated self-heal scripts (restart services, clear caches, reboot stuck agents).
  6. Document alert definitions and create standardized alert templates per client.
Goal: signal > noise Lean: remove non-value steps
RMM 2. Explain your approach to patch management across multiple clients
  1. Segment devices by OS, department, and business impact.
  2. Create staggered patch windows to reduce simultaneous impact.
  3. Test critical patches in controlled pilot groups first.
  4. Review vendor advisories for high-risk vulnerabilities.
  5. Use RMM automation to enforce reboots, verify install, and generate compliance reports.
  6. Document exceptions and remediation timelines.
ITIL: Change mgmt for high-risk patches Security: CVE mitigation
RMM 3. How do you build a proactive monitoring strategy?
  1. Implement baseline monitors: CPU, RAM, disk, processes, services, event logs, AV, backups.
  2. Add custom monitors for client-specific apps (databases, ERP, point-of-sale, etc.).
  3. Configure uptime/heartbeat monitoring and recovery scripts.
  4. Integrate RMM → PSA ticketing for automated incident creation.
  5. Set up monthly health reports and automatic remediation tasks.
Goal: detect before users notice Proactive > reactive
RMM 4. Give examples of automation scripts you’ve deployed in an MSP environment
  1. Automated service restarts for stuck Windows services.
  2. Temp file cleanup, log rotation, cache clearing.
  3. Automated AD password expiry notifications.
  4. Printer spooler repair scripts.
  5. Patch retry workflows for failed updates.
  6. Disk space remediation (cleanup → alert → escalate).
PowerShell mastery = MSP efficiency
M365 1. How do you troubleshoot Exchange Online mail flow failures?
  1. Check Message Trace for delivery status codes.
  2. Verify SPF, DKIM, DMARC records.
  3. Review transport rules & spam filtering policies.
  4. Inspect mailbox forwarding, delegates, or blocked senders.
  5. Check Exchange connector health (hybrid only).
  6. Run remote connectivity analyzer tests.
Common issue → SPF softfail Fix → update DNS, revalidate
M365 2. Troubleshooting Microsoft Teams authentication issues
  1. Check Azure AD sign-in logs for MFA/conditional access blocks.
  2. Clear Teams cache or reinstall local app.
  3. Validate device compliance (Intune).
  4. Review network proxy/firewall for Teams endpoints.
  5. Check service status in Microsoft 365 Admin Center.
Teams issues = 75% auth or network
M365 3. How do you troubleshoot SharePoint/OneDrive access or sync issues?
  1. Check SharePoint site permissions & role groups.
  2. Validate file-level permissions (unique permission blocks).
  3. Check OneDrive sync client health & version.
  4. Ensure correct tenant sign-in (multi-tenant issue common).
  5. Inspect sensitivity labels or DLP policies blocking upload.
Users see “Access Denied”? → permission inheritance broken
M365 4. Troubleshooting Intune device enrollment & compliance
  1. Verify user licensing (Intune + Azure AD P1).
  2. Check device limits per user.
  3. Inspect enrollment restrictions and compliance policies.
  4. Confirm the device meets OS/hardware requirements.
  5. Review MDM authority & Autopilot profile assignments.
Intune failures → 90% licensing or MDM conflicts
AD 1. How do you troubleshoot domain login failures?
  1. Check user account status (locked/disabled/expired password).
  2. Verify network connectivity to domain controllers.
  3. Review DC event logs for Kerberos/NTLM failures.
  4. Check time synchronization (Kerberos requires <5 mins drift).
  5. Inspect DNS SRV records for missing DC entries.
Time skew → instant authentication failures
AD 2. Troubleshooting GPO not applying
  1. Run gpresult /r to see applied policies.
  2. Check GPO link order & inheritance.
  3. Verify security filtering & WMI filters.
  4. Check SYSVOL replication (DFSR issues).
  5. Ensure client can reach DC over SMB/LDAP/Kerberos.
SYSVOL replication = root of many GPO issues
AD 3. Troubleshooting file share and NTFS permission issues
  1. Check share permissions vs NTFS ACLs (lowest privilege wins).
  2. Verify user group membership.
  3. Inspect broken inheritance or explicit denies.
  4. Check offline files and DFS namespace redirection.
  5. Review file locks and open sessions.
Explicit DENY → overrides everything
AD 4. How do you troubleshoot print server issues?
  1. Restart print spooler and clear stuck jobs.
  2. Update printer drivers and check compatibility.
  3. Verify correct driver model (Type 3 preferred).
  4. Check GPO-deployed printers for mapping issues.
  5. Review print server event logs for queue corruption.
Printer issues = 70% spooler corruption
Security 1. How do you respond to a suspicious login or MFA bypass attempt?
  1. Review Azure AD sign-in logs for location/device anomalies.
  2. Force password reset; revoke sessions.
  3. Enable/verify MFA and Conditional Access baseline policies.
  4. Check device compliance and remove unmanaged devices.
  5. Review mailbox forwarding rules for suspicious activity.
Impossible travel = red flag
Security 2. Troubleshooting Conditional Access blocking legitimate users
  1. Check policy assignment scope and conditions.
  2. Review “Grant” controls (e.g., MFA, device compliance).
  3. Inspect sign-in logs for failed requirement.
  4. Create temporary exclusion group for testing.
  5. Adjust or split policies using least privilege and least complexity.
Always test CA policies in staging
Scenario 1. SEV1 – Global Microsoft 365 outage handling
  1. Check service health dashboard & Twitter/X engineering feed.
  2. Confirm scope: global, regional, tenant-specific.
  3. Communicate to leadership + clients with ETA and workaround.
  4. Adjust workload routing where possible (e.g., cached mode).
  5. Create master incident ticket & status page updates.
Communication = critical path
Scenario 2. Domain Controller failure or AD outage
  1. Check DC services (NTDS, DNS, Netlogon).
  2. Confirm SYSVOL/NETLOGON share availability.
  3. Check replication health (repadmin /replsummary).
  4. Verify event logs for USN rollback / tombstone issues.
  5. Force replication or seize FSMO roles if necessary.
Scenario 3. Diagnosing complete network outage
  1. Check firewall WAN state & ISP link quality.
  2. Check whether DHCP/DNS services are down internally.
  3. Isolate: WAN-only? LAN-only? Wi-Fi-only?
  4. Check switching loops (STP blocking ports).
  5. Identify problem device and isolate physically.
Ops 1. How do you prevent recurring incidents across client environments?
  1. Trend analysis of incidents → find patterns.
  2. Convert repeat issues into automation runbooks.
  3. Deploy improved monitoring to detect earlier.
  4. Perform root cause analysis & document fixes.
  5. Standardize configurations across all clients.
ITIL: Continual Improvement
Ops 2. Explain your approach to environment documentation at scale
  1. Create documentation templates (network maps, AD structure, backup plans).
  2. Document changes in real-time (ITIL Change Management).
  3. Version-control documentation in SharePoint/Confluence.
  4. Use diagrams (Visio / draw.io) for clarity.
  5. Ensure onboarding/offboarding workflows documented.
Documentation saves hours
